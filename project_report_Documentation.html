1. Problem Statement

The Challenge

Traditional pet adoption processes are often fragmented, opaque, and inefficient. Potential adopters face several critical hurdles:

  • Lack of Information: Incomplete pet profiles without detailed personality traits or history.
  • Complex Procedures: Paper-based, slow application processes with no visibility into status.
  • Poor User Experience: Outdated websites that are difficult to navigate on mobile devices.
  • Trust Issues: Uncertainty about the legitimacy of shelters or the health status of pets.

The Solution

Pet Adoption Premium is a modern, full-stack web application designed to streamline the adoption journey. It connects ethically raised pets with loving homes through a premium, transparent, and user-friendly digital experience.

Key Objectives:

  • Transparency: Detailed pet profiles including likes, dislikes, and favorite games.
  • Efficiency: Digital adoption requests with real-time status tracking.
  • Accessibility: A fully responsive mobile-first design.
  • Administrative Control: A robust admin dashboard to manage pets, users, and requests efficiently.

2. System Architecture & Database Schema

The application is built on the MERN Stack (MongoDB, Express.js, React, Node.js).

Database Schema Diagram

The following Entity-Relationship Diagram (ERD) represents the MongoDB collections and their relationships.

erDiagram User ||--o{ AdoptionRequest : makes Pet ||--o{ AdoptionRequest : receives User { ObjectId _id PK String name String email String password Boolean isAdmin Date createdAt Date updatedAt } Pet { ObjectId _id PK String name String type String breed Number age String description String[] likes String[] dislikes String[] games String imageUrl Number price Boolean isAdopted Date createdAt Date updatedAt } AdoptionRequest { ObjectId _id PK ObjectId user FK ObjectId pet FK String message String status "Pending, Approved, Rejected" Date createdAt Date updatedAt }

Schema Description

  1. User Collection: Stores user credentials and role (Admin/Regular).
  2. Pet Collection: Contains detailed information about adoptable pets, including adoption status (isAdopted flag).
  3. AdoptionRequest Collection: Acts as the link between a User and a Pet, storing the application message and current approval status.

3. Technology Stack

Frontend

  • React.js: Component-based UI library.
  • Vite: Next-generation build tool for fast development.
  • Tailwind CSS: Utility-first CSS framework for rapid, custom styling.
  • Framer Motion: Library for production-ready animations.
  • Axios: Promise-based HTTP client for API requests.

Backend

  • Node.js: JavaScript runtime environment.
  • Express.js: Web application framework for API routing.
  • MongoDB & Mongoose: NoSQL database and Object Data Modeling (ODM) library.
  • JWT (JSON Web Tokens): Secure user authentication.

Deployment

  • Railway: Cloud infrastructure for hosting both backend and frontend.
